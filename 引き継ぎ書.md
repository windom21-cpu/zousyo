# K-books 引き継ぎ書

このプロジェクトを後から触る(自分含む)ときの最初の参照先。仕様の「なぜそうしたか」を中心にまとめる。コードを読めばわかる「何をしているか」は最低限に。

## 1. 概要

- **目的**: コミック蔵書管理。買物中に重複買いを防ぐ
- **想定ユーザー**: 自宅内の家族数人(同時更新は稀)
- **公開URL**: https://windom21-cpu.github.io/k-books/
- **リポジトリ**: https://github.com/windom21-cpu/k-books (Public)
- **GitHubユーザー**: windom21-cpu

## 2. アーキテクチャ(ひと言)

> GitHub Pages で配信する素のHTML/JSが、`data.json` を **GitHub Contents API 経由で直接読み書きする**。サーバ・DBなし、無料、PAT を家族で共有して書き込み権限を渡す。

```
[ ブラウザ(家族端末) ]
   ├── 閲覧: raw.githubusercontent.com から data.json
   └── 書込: api.github.com/repos/.../contents/data.json + PAT (fine-grained)
```

## 3. データ構造(data.json)

```json
{
  "items": [
    {
      "id": "uuid",
      "series": "シリーズ名(タイトルから抽出)",
      "seriesYomi": "読み仮名(openBDのcollationkey)",
      "volume": 13,
      "edition": "" | "新装" | "愛蔵" | "完全" | "文庫" | "その他",
      "title": "openBD原文",
      "author": "...",
      "publisher": "...",
      "isbn": "9784...",
      "coverUrl": "https://...",
      "addedBy": "ニックネーム",
      "note": "",
      "acquiredAt": "YYYY-MM-DD"
    }
  ]
}
```

**重複判定キー**: ISBN優先 → なければ `normalize(series) | volume | normalize(edition)`

## 5. core.js が抱えている関心事

| 関数 | 責務 |
|---|---|
| `bootstrapTokenFromHash()` | URL `#token=...` を localStorage に取り込み即除去 |
| `getPAT/setPAT/getNick/setNick` | localStorage アクセス |
| `getPATSetAt()` | この端末でPATを保存した日付(`kbooks_pat_set_at`)を返す |
| `migrateLegacyKeys()` | `zousyo_*` → `kbooks_*` 旧キー移行 |
| `fetchData()` | PAT有無で API/raw を切替えて data.json 取得 |
| `commitMutation(mutate, msg)` | SHA比較で安全に PUT、競合6回まで指数バックオフ。直前PUTのレスポンスSHAを `_cache` に残し連続書込のレプリカ遅延を回避 |
| `lookupISBN(isbn)` | openBD `/v1/get` |
| `findDuplicate` | ISBN優先 |
| `guessSeriesFromTitle` | タイトル末尾の巻数表記を剥ぐ |
| `findExistingSeries` | 正規化後一致する既存シリーズ名の正規表記を返す(表記ゆれ寄せ) |
| `normalize` | NFKC + 小文字 + 空白除去 + 記号除去 |
| `parseVolume` | "第3巻" "(03)" 等から数値抽出 |
| `attachCalendarPicker(textId, pickerId)` | テキスト入力欄に、上にオーバーレイされた `<input type=date>` の change を反映させる(focus時にテキスト値で初期化、change時に書戻し) |

## 6. 重要な設計判断と理由

### 6.1 認証
- **公開リポ + 共有PAT** にした理由: 家族のみが知るURLにPATを埋めたQRを配布する運用なら、書込権限の制限と運用簡便さを両立できる
- 万一QRが流出しても、fine-grained PAT を `Contents:RW` のみ・対象1リポに絞ってあるため最大被害は data.json の改竄(git revert で復旧可)

### 6.2 シリーズ名はタイトルから抽出
- openBDの `summary.series` は **レーベル名**(例: "ジャンプコミックス")を返す。これを使うと別作品同士が同名扱いになり重複判定が崩壊する
- そのため `guessSeriesFromTitle(title)` でタイトル末尾の巻数を剥ぐ → これを `series` フィールドに入れる
- それでもブレるので **保存時に `findExistingSeries` で既存表記に寄せる**

### 6.3 重複判定はISBN優先
- 物理書籍はISBNで一意。シリーズ名のブレに影響されない
- ISBN欠落時のみ `series + volume + edition` フォールバック

### 6.4 表示の柔軟マッチ
- 検索画面のシリーズグルーピングは `normalize()` キーで集約
- 末尾スペース・全/半角・記号差は同一視。意味的に違う表記(略称/別名)は別物のまま

### 6.5 連続書込の競合対策
- GitHub の Contents API は読み取りレプリカに反映遅延がある
- PUT成功時のレスポンス `content.sha` を `_cache` に保持し、次回PUTでは再GETせずキャッシュSHAを使う
- 他者が同時に書いた場合だけキャッシュが無効になり、指数バックオフで再取得・再試行

### 6.6 ブラウザキャッシュ対策
- 全HTMLに `Cache-Control: no-cache` メタタグ
- アセット参照URLに `?v=X.Y` クエリ
- バージョン上げで自動的に新URL扱い → ブラウザは再取得
- JSモジュールの内部import (`from './core.js?v=X.Y'`) も同じバージョンに揃える
- 取りこぼし防止のため `bump.sh` で一括書換

### 6.7 カレンダーピッカーのオーバーレイ方式
- 当初は📅ボタンのclickで `<input type=date>` を動的createElement → `showPicker()` していた
- Android Chrome の一部環境で発火しないことが判明
- 現方式: 永続的な `<input type=date opacity:0>` を📅ボタンの上に絶対配置。タップは透明なinputに直接届き、各OSのネイティブカレンダーが開く
- スタイルは `app.css` の `.date-pick-wrap`

### 6.8 一括登録キューのlocalStorage永続化
- スマホで連続スキャン中にブラウザがバックグラウンド殺傷されると、未登録キューが消えるリスク
- `bulk.js` は `render()` 内で `kbooks_bulk_queue` に毎回保存、起動時に復元
- 復元時は既存蔵書と再突合(その間に他端末で登録された分は最新に対して再判定)

### 6.9 シリーズ統合は手動UI
- 自動類似性検出(Levenshtein等)も検討したが、誤マージのリスクとUI複雑化を嫌った
- 設定画面で 統合元/統合先 を2つのプルダウンから選んで統合する方式
- 「ジョジョ」と「ジョジョの奇妙な冒険」のような略称/正式名の手動マージ用途

### 6.10 変更履歴は閲覧のみ
- 設定画面の「変更履歴」は `/repos/{owner}/{repo}/commits?path=data.json` を読むだけ
- アプリ内revertは実装していない: 取消は `git revert` で行う想定
- 各行のSHAリンクから GitHub のコミット詳細画面に飛んで diff を確認できる

## 7. バージョン上げ手順

リポジトリ直下の `bump.sh` を使う。

```bash
./bump.sh 2.1
git add -A && git commit -m "..." && git push
```

`bump.sh` は `assets/config.js` の現バージョンを読み取り、全HTML/JS の `?v=X.Y` と `version` 文字列を新値に書換える。1ファイル抜け事故を防ぐ。

## 9. PAT運用

### 発行
1. GitHub → Settings → Developer settings → Fine-grained tokens
2. Repository: `k-books` のみ
3. Permissions: `Contents: Read and write`
4. 期限: 任意(現状無期限を選択中)

### 失効・流出対応(端末紛失含む)
家族の誰かが端末を紛失したり、QRが流出した可能性がある場合の手順:

1. **即時**: GitHub → Settings → Developer settings → Fine-grained tokens → 該当PATを **Revoke**
   - これで紛失端末からの書込みは止まります(localStorageにあるPATは無効化される)
   - 閲覧は引き続き可能(公開リポなので)
2. 新しいPATを同じ条件で発行(Repository: k-books, Contents: RW)
3. 自分のメイン端末の設定画面でPATを差替え(古い値をクリアして新値を保存)
4. 「家族招待用QR」を再生成 → 信頼できる家族に再配布
5. 念のため `gh api repos/windom21-cpu/k-books/commits` で直近のコミットを確認、不審な書込があれば `git revert` で戻す

### 「どの端末がPATを持っているか」を可視化したいとき
今は端末別に管理していないため、誰の端末で何が起きたかは追えません。家族が増えたり管理を厳密にしたいときは「家族ごとに別PAT」運用への切替を検討。各人のPATは設定画面でnickをセット → addedBy が記録されるため、誰が登録したかは追える状態にはなる。

## 11. 既知の制限

- **MS UI ゴシックはWindowsのみ** → モバイルではフォールバック(retro感は家のPCのみ)
- **カメラはHTTPS必須** → GitHub PagesはOK、`file://`不可
- **GitHub Pages無料枠でPrivate repo不可** → データは公開状態(蔵書情報なので実害は薄い)
- **同時書込時の競合** → 6回バックオフリトライ。家族の同時操作はまずないので実害は出ていない
- **PAT流出時はRevoke頼り** → 紛失端末を特定する手段なし(セクション9参照)

## 13. トラブルシューティング(私の経験的)

| 症状 | 確認順序 |
|---|---|
| 登録が反映されない | ① タイトルバーのバージョンが最新か → ② 設定画面でPAT設定済か → ③ コンソールにエラーないか |
| 古いUIが表示される | ハードリロード(Ctrl+Shift+R)・DevToolsのDisable Cache |
| PUT失敗 401/403 | PATが失効・権限不足。Contents:RW を確認 |
| PUT失敗 422/409 連発 | 他端末と競合中、または読み取りレプリカ遅延。少し待って再試行 |
| openBDで書誌が見つからない | 古い本・同人誌は未収録。一括登録なら編集ボタンで手入力で補える |
| QRが描画されない | 設定画面のCDN参照を確認(`qrcode` は esm.sh 経由でなければ動かない) |
| 📅ボタンを押してもピッカーが開かない | `<input type=date>` の オーバーレイがボタンの上にあるか(.date-pick-wrap CSSが適用済か)を確認 |
| 一括登録キューがリロードで消える | `localStorage` を見て `kbooks_bulk_queue` の有無を確認。プライベートウィンドウだと永続化されない |
| シリーズ統合が反映されない | 設定画面でリロード前後の data.json を確認。`merge: ...` のコミットが履歴に出ているか |
