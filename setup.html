セットアップ手順

K-books を使い始めるための手順です。書き込み(登録・編集・削除)を行う端末ごとに必要です。閲覧だけなら何も設定不要です。

このサイト
アプリURLhttps://windom21-cpu.github.io/k-books/
リポジトリgithub.com/windom21-cpu/k-books

STEP 1 — fine-grained PAT を発行

GitHubにログインした状態で、こちらを開く。または:

右上のアイコン → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token

以下のように設定:

項目値
Token namek-books-write(任意)
Expiration1 year(最長)
Resource ownerwindom21-cpu
Repository accessOnly select repositories → k-books
Repository permissionsContents: Read and write

「Generate token」を押下 → github_pat_xxxxxx... が表示されるのでコピー。再表示できないのでメモ帳等に一時保存。

期限は最長1年。失効したら同じ手順で作り直し → 設定画面で更新。カレンダーに登録推奨。

STEP 2 — 設定画面でPATを保存

  1. トップページを開く
  2. 「設定」をクリック
  3. PAT欄にSTEP1でコピーしたトークンを貼り付け
  4. ニックネームを入力(任意・登録履歴に残ります)
  5. 「保存」を押下

これでこの端末から書き込み(登録・編集・削除)が可能になります。

STEP 3 — 家族の端末に招待

家族ごとに番号(002, 003 …)が自動採番されます。あなたは001。

  1. あなたの端末で「設定」→「家族招待用QR」→「QR表示」
  2. 表示されたQRを家族のスマホのカメラ等で読み取る
  3. 家族の端末でアプリが開き、「PATを受け取りました(ニックネーム: 002)」と表示される
  4. これで家族の端末からも書き込み可能になる(ニックネームも自動セット済)

「ホームに追加」(PWA)で使う場合の注意: PWAから起動した端末はブラウザとは別の保存領域になるため、上記の方法ではPATが渡りません。その場合は PWAを起動 → 設定 → 「招待QRを読み取る」 → スキャン開始 でこちらから読み取ってください。

注意: QRはPATそのものと同等の重みです。SNS掲載・来客の見える場所への掲示は避けてください。万一漏洩したらSTEP1のページでPATを失効させ、新しいPATを発行 → 再配布。

STEP 4 — 動作確認

  • トップ: 「登録冊数: 0 冊 / PAT: 設定済」と表示される
  • 登録(手動): ISBN手入力 → 「openBD取得」で自動補完 → 保存
  • 登録(バーコード): スマホで「スキャン開始」→ コミックのバーコードを向ける → 自動でフォーム補完 → 保存
  • 検索: 「わんぴーす」「ONE」など部分入力 → サジェスト表示、所持最大巻と抜け巻が表示される
  • 重複チェック: 同シリーズ・同巻・同版を登録しようとすると警告が出る

背面カメラが選ばれない / フロントが映る自動で背面→フロント→列挙順に再試行。PC等デュアルカメラ未対応端末は仕方なし
連続スキャンで次のバーコードが認識されないv2.7以降はEAN-13限定+BarcodeDetector優先で大幅高速化。それでも遅いなら ① ブラウザを最新版に ② ピントが合うようバーコードを20cmほど離す ③ カメラレンズを清掃
登録(バーコード)後に手入力欄が空のままopenBDの応答待ち。数秒待つ。それでも空なら openBD未収録(古本・同人誌等)。手入力で補完して保存
登録・検索
登録直後に検索で出ないGitHubの反映に数秒。検索画面の「再読込」を押下、または検索画面に再度入り直す
同じシリーズが微妙に別シリーズ扱いになる登録時の表記揺れ。設定 → 「シリーズ統合・改名」で集約。複数の表記をまとめる場合は「複数選択する」を使用
openBDで書誌が見つからない古い本・同人誌は未収録。一括登録なら「編集」ボタンから手入力で補える
「最近の登録」が同日内で並びが期待と違う登録順(配列末尾)で並ぶ。トップを再アクセスかF5で最新化される
UI・表示
古いUIが表示されるハードリロード(Ctrl+Shift+R)。または DevTools → Network → Disable cache
📅ボタンを押してもピッカーが開かない透明な<input type=date>がボタン上にオーバーレイされている。CSSが当たっていない場合は再読込
