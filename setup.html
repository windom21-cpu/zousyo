★ K-books セットアップ ★

K-books を使い始めるための手順です。
書き込み(登録・編集・削除)を行う端末ごとに必要です。閲覧だけなら何も設定不要です。

★ STEP 1 ★ fine-grained PAT を発行

GitHubにログインした状態で、こちらを開く。

右上のアイコン → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token

設定項目 値
Token name k-books-write(任意)
Expiration 1 year(最長)
Resource owner windom21-cpu
Repository access Only select repositories → k-books
Repository permissions Contents: Read and write

「Generate token」を押下すると github_pat_xxxxxx... が表示されるのでコピー。
再表示できないのでメモ帳等に一時保存。

※ 期限は最長1年。失効したら同じ手順で作り直し → 設定画面で更新。カレンダーに登録推奨。


★ STEP 2 ★ 設定画面でPATを保存

1. トップページを開く
2. 「設定」をクリック
3. PAT欄にSTEP1でコピーしたトークンを貼り付け
4. ニックネームを入力(任意・登録履歴に残ります)
5. 「保存」を押下

これでこの端末から書き込み(登録・編集・削除)が可能になります。


★ STEP 3 ★ 家族の端末に招待

家族ごとに番号(002, 003 …)が自動採番されます。あなたは001。

1. あなたの端末で「設定」→「家族招待用QR」→「QR表示」
2. 表示されたQRを家族のスマホのカメラ等で読み取る
3. 家族の端末でアプリが開き、「PATを受け取りました(ニックネーム: 002)」と表示される
4. これで家族の端末からも書き込み可能になる(ニックネームも自動セット済)

「ホームに追加」(PWA)で使う場合の注意
PWAから起動した端末はブラウザとは別の保存領域になるため、上記の方法ではPATが渡りません。
その場合は PWAを起動 → 設定 → 「招待QRを読み取る」 → スキャン開始 でこちらから読み取ってください。

※ QRはPATそのものと同等の重みです。SNS掲載・来客の見える場所への掲示は避けてください。万一漏洩したらSTEP1のページでPATを失効させ、新しいPATを発行 → 再配布。


★ STEP 4 ★ 動作確認

項目 確認内容
トップ 「登録冊数: 0 冊 / PAT: 設定済」と表示される
登録(手動) ISBN手入力 → 「openBD取得」で自動補完 → 保存
登録(バーコード) スマホで「スキャン開始」→ コミックのバーコードを向ける → 自動でフォーム補完 → 保存
検索 「わんぴーす」「ONE」など部分入力 → サジェスト表示、所持最大巻と抜け巻が表示される
重複チェック 同シリーズ・同巻・同版を登録しようとすると警告が出る

★ トラブルシューティング ★

症状 対処
― PAT・招待まわり ―
ホーム追加した端末(PWA)で「PAT: 未設定」になる PWAはブラウザと別ストレージのため、PWA内で改めて受信する必要あり。
設定 → 「招待QRを読み取る」 → スキャン開始 → 管理者のQRを読み取り
招待QRをスキャンしてもPATが入らない (1) タイトルバーで現バージョンが最新か確認
(2) F12コンソールに「PAT received via URL」が出ているか
(3) 古いiOSではフラグメント方式のQRが落ちることあり、最新版を使えばクエリ方式で動く
保存時「PATが未設定です」 設定画面でPATを保存(or 招待QR読取)
保存失敗 401/403 PATの権限不足または失効。STEP1で再発行 → 設定で更新
保存失敗 422/409 が繰り返す 他端末と同時書込中、または GitHub読取レプリカの遅延。自動で6回リトライするが、それでもダメなら時間をおいて再試行
― カメラ・スキャン ―
カメラが起動しない / 「カメラ起動失敗」 (1) HTTPSでアクセス(GitHub PagesならOK、file://不可)
(2) カメラ権限を許可(初回ブロックすると要リセット: ブラウザのサイト設定→カメラ→許可)
(3) iOS Safari / Android Chrome を推奨
(4) 他アプリ(LINEなど)がカメラ占有していないか
背面カメラが選ばれない / フロントが映る 自動で背面→フロント→列挙順に再試行。PC等デュアルカメラ未対応端末は仕方なし
連続スキャンで次のバーコードが認識されない v2.7以降はEAN-13限定+BarcodeDetector優先で大幅高速化。それでも遅いなら (1) ブラウザを最新版に (2) ピントが合うようバーコードを20cmほど離す (3) カメラレンズを清掃
登録(バーコード)後に手入力欄が空のまま openBDの応答待ち。数秒待つ。それでも空なら openBD未収録(古本・同人誌等)。手入力で補完して保存
― 登録・検索 ―
登録直後に検索で出ない GitHubの反映に数秒。検索画面の「再読込」を押下、または検索画面に再度入り直す
同じシリーズが微妙に別シリーズ扱いになる 登録時の表記揺れ。設定 → 「シリーズ統合・改名」で集約。複数の表記をまとめる場合は「複数選択する」を使用
openBDで書誌が見つからない 古い本・同人誌は未収録。一括登録なら「編集」ボタンから手入力で補える
「最近の登録」が同日内で並びが期待と違う 登録順(配列末尾)で並ぶ。トップを再アクセスかF5で最新化される
表紙画像が表示されない (1) ISBN登録時に openBD/Google Books どちらにもcoverが無かった可能性
(2) 編集モーダルで「再取得」を試す
(3) それでもダメなら手動でURL貼付け(Amazon商品画像URL等)
― UI・表示 ―
古いUIが表示される ハードリロード(Ctrl+Shift+R)。または DevTools → Network → Disable cache
📅ボタンを押してもピッカーが開かない 透明な<input type=date>がボタン上にオーバーレイされている。CSSが当たっていない場合は再読込
QRが描画されない 設定画面のCDN参照を確認(qrcode は esm.sh 経由)
一括登録キューがリロードで消えた localStorageに保存している。プライベートウィンドウだと永続化されない場合あり
