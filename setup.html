セットアップ手順

K-books を使い始めるための手順です。書き込み(登録・編集・削除)を行う端末ごとに必要です。閲覧だけなら何も設定不要です。

このサイト
アプリURLhttps://windom21-cpu.github.io/k-books/
リポジトリgithub.com/windom21-cpu/k-books

STEP 1 — fine-grained PAT を発行

GitHubにログインした状態で、こちらを開く。または:

右上のアイコン → Settings → Developer settings → Personal access tokens → Fine-grained tokens → Generate new token

以下のように設定:

項目値
Token namek-books-write(任意)
Expiration1 year(最長)
Resource ownerwindom21-cpu
Repository accessOnly select repositories → k-books
Repository permissionsContents: Read and write

「Generate token」を押下 → github_pat_xxxxxx... が表示されるのでコピー。再表示できないのでメモ帳等に一時保存。

期限は最長1年。失効したら同じ手順で作り直し → 設定画面で更新。カレンダーに登録推奨。

STEP 2 — 設定画面でPATを保存

  1. トップページを開く
  2. 「設定」をクリック
  3. PAT欄にSTEP1でコピーしたトークンを貼り付け
  4. ニックネームを入力(任意・登録履歴に残ります)
  5. 「保存」を押下

これでこの端末から書き込み(登録・編集・削除)が可能になります。

STEP 3 — 家族の端末に招待

  1. あなたの端末で「設定」→「QR表示」
  2. 表示されたQRを、家族のスマホのカメラで読み取る
  3. 家族の端末でアプリが開き、「PATを受け取りました」と表示される
  4. これで家族の端末からも書き込み可能になる

注意: QRはPATそのものと同等の重みです。SNS掲載・来客の見える場所への掲示は避けてください。万一漏洩したらSTEP1のページでPATを失効させ、新しいPATを発行 → 再配布。

STEP 4 — 動作確認

  • トップ: 「登録冊数: 0 冊 / PAT: 設定済」と表示される
  • 登録(手動): ISBN手入力 → 「openBD取得」で自動補完 → 保存
  • 登録(バーコード): スマホで「スキャン開始」→ コミックのバーコードを向ける → 自動でフォーム補完 → 保存
  • 検索: 「わんぴーす」「ONE」など部分入力 → サジェスト表示、所持最大巻と抜け巻が表示される
  • 重複チェック: 同シリーズ・同巻・同版を登録しようとすると警告が出る
